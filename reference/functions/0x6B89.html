0x6B89 — Diagnostic Mode Initializer

Field Value
Address 0x6B89
Size 56 bytes
Blocks 7
Type subroutine
Callers 1 (JSR from 0x97D4; fall-through from 0x6B60 not counted as a caller by flood-fill)
Confidence confirmed

Summary

Toggles diagnostic/test mode based on 0x305C bit 1 and initializes diagnostic state when transitioning. When entering test mode (0x305C bit 1 set and 0x3510 == 0x00), sets 0x3510 to 0xFF and marks pending faults at 0x31CF bits 4,2,7 (mask 0x94). When leaving test mode (0x305C bit 1 clear and 0x3510 == 0xFF), clears 0x3510 to 0x00. On any mode transition, clears all diagnostic accumulators: fault flag registers (0x332E, 0x3330), diagnostic counters (0x3506, 0x3508), and initialization state variables (0x3504, 0x3512, 0x3513).

Inputs

  • 0x305C bit 1: diagnostic mode request (from mode_select_flags)
  • 0x3510: current diagnostic mode state (0x00 = normal, 0xFF = test)

Outputs

  • 0x3510: updated diagnostic mode state
  • 0x31CF: pending fault bits set (bits 2, 4, 7 = mask 0x94) on test mode entry
  • 0x332E: cleared to 0x0000 (16-bit)
  • 0x3330: cleared to 0x0000 (16-bit)
  • 0x3506: cleared to 0x0000 (16-bit)
  • 0x3508: cleared to 0x0000 (16-bit)
  • 0x3504: cleared to 0x00
  • 0x3512: cleared to 0x00
  • 0x3513: cleared to 0x00

RAM Reads

  • 0x305C: mode_select_flags
  • 0x3510: control_reg_3510

RAM Writes

  • 0x3510, 0x31CF, 0x332E, 0x3330, 0x3506, 0x3508, 0x3504, 0x3512, 0x3513
