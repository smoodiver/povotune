0x6080 — Map Lookup (Bilinear Interpolation)

Field Value
Address 0x6080 (primary), 0x608D (alternate), 0x6093 (8x8 mode)
Size 13 bytes (0x6080–0x608C)
Blocks 1 (entry stub only — shared body attributed to sub_6093 by flood-fill)
Type subroutine (three entry points into shared body)
Callers 4 via 0x6080 (8 total across all entry points: +1 via 0x608D, +3 via 0x6093)
Confidence confirmed

Summary

Core 2D bilinear interpolation engine used by all major ECU subsystems. Looks up a value from a 16x16 or 8x8 ROM table indexed by RPM and engine load, interpolating between the four surrounding cell values for smooth output. Three entry points select the operating mode: 0x6080 for 16x16 tables, 0x6093 for 8x8 tables, and 0x608D for an alternate mode that skips load-axis interpolation. The caller sets up table and axis pointers in workspace RAM before calling; the result is returned in the A register.

Inputs

  • 0x3072: pointer to map table start in ROM
  • 0x3074: pointer to RPM axis breakpoint table (16 or 8 entries)
  • 0x3076: pointer to load axis breakpoint table (16 or 8 entries)
  • 0x307A: current RPM value (raw timer period >> 2, from 0x3062)
  • 0x3078/0x3079: current load value (0x313C << 3)

Outputs

  • A register: interpolated table value (8-bit)
  • 0x3085: combined 2D table index (column + row x width)
  • 0x3077: load-axis interpolated value (intermediate result, row 1)
  • 0x3076: load-axis interpolated value (intermediate result, row 2)
  • 0x307A: clamped RPM value (may be updated if RPM exceeds axis range)
