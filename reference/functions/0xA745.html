0xA745 — Timing Correction Decay

Field Value
Address 0xA745
Size 42 bytes (0xA745–0xA76E)
Blocks 8
Type subroutine
Callers 1 — knock_control_dispatcher (0xA683 at 0xA6DF)
Confidence confirmed

Summary

Decays the knock timing correction value ($31B8) toward zero or applies a fixed default. Behavior depends on the knock correction enable flags:

  1. Knock correction fully enabled ($31C2 = 0xFF): Loads the ROM default timing correction from $4D9B directly into $31B8, bypassing any decay.
  2. Sensor active ($31C1 nonzero): Adds a ROM step value ($4D90) to the current timing correction, clamping to a maximum from $4D92. This gradually increases the correction (recovery toward baseline).
  3. Neither active: Returns immediately with no changes.

The result is also mirrored to $01B8 (likely a diagnostic/SSM output mirror).

Disassembly

A745  BBC $31C2,#$FF,$A74F ; if knock_corr_enable != 0xFF, -> check sensor
A74A  LDX $4D9B           ; load default correction from ROM
A74D  BRA $A767           ; -> store and return
A74F  BBC $31C1,#$FF,$A76E ; if sensor_activity = 0, -> return (no change)
A754  CLM                 ; 16-bit mode
A755  LDA $31B8           ; load current timing_correction
A758  CLC
A759  ADC $4D90           ; add ROM step value
A75C  BCS $A764           ; if overflow, -> clamp to max
A75E  TAX
A75F  CPX $4D92           ; compare against maximum
A762  BCC $A767           ; if below max, keep value
A764  LDX $4D92           ; clamp to maximum
A767  STX $31B8           ; store updated correction
A76A  STX $01B8           ; mirror to diagnostic output
A76D  SEM                 ; back to 8-bit
A76E  RTS

Inputs

  • $31C2 (knock_corr_enable) — 0xFF = fully enabled, controls dispatch
  • $31C1 (sensor_activity_flags) — nonzero = sensor active, controls decay path
  • $31B8 (timing_correction) — current correction value (16-bit)

Outputs

  • $31B8 (timing_correction) — updated correction value
  • $01B8 — diagnostic mirror of timing_correction
