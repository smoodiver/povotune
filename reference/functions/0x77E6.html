0x77E6 — MIL Lamp Driver

Field Value
Address 0x77E6
Size ~322 bytes (0x77E6–0x7927)
Blocks 70
Type subroutine
Callers 1 (health_check_dispatcher 0x6B14, slot 25)
Callees 0x7928, 0x79C7, 0x7A16, 0x6346, 0x634E, 0x920B
Confidence confirmed

Summary

Controls the Malfunction Indicator Lamp (MIL / Check Engine Light) on the instrument cluster. Manages three distinct operating modes: self-test blink code output, normal-mode steady illumination, and a test-mode fault code readout pattern. Uses port 0x2000 bit 5 (via shadow register 0x35A5) to drive the lamp.

The function has several major phases:

Phase 1 — Timer countdown (0x77E6–0x780D): Decrements two countdown timers (0x3190, 0x3191) via saturating_decrement (0x6346). If 0x35A4 bit 0 (self-test mode) is active and timer 0x3190 has reached zero, reloads it with 0x27 (39 ticks) and toggles port 0x2000 bits 0+5 (EOR #$21) — this creates the blink pattern for the self-test indicator. Returns early while self-test is active.

Phase 2 — Mode dispatch (0x780E–0x781E): Three-way branch based on system state:

  • If 0x3510 is non-zero (test mode active): jump to Phase 4 (test-mode handler)
  • If 0x305C bit 2 is set (clear-codes mode): jump to Phase 3 (clear-codes handler)
  • Otherwise: fall through to Phase 2b (normal mode)

Phase 2b — Normal mode (0x781E–0x7878): If blink timers (0x3191, 0x3192) are still running, returns (maintains current lamp state). Otherwise checks for active fault codes in 0x332E/0x3330. If faults present and persistent fault counter (0x3216) >= 0x0494 (1172), increments fault persistence counter (0x318E) via saturating_increment_y; when >= 0x62 (98), forces lamp ON. If counter < 0x0494, sets 0x318E to 0xFFFF and forces lamp ON. If no faults present, clears 0x318E, reloads timer (0x3190 = 0x23/35 ticks), and sets lamp state based on 0x3051 bit 2 (engine running indicator).

Phase 3 — Clear-codes mode (0x7879–0x78C6): Checks latched fault registers (0x350A, 0x350C). If faults exist, enters a multi-phase blink sequence: calls 0x7928 (fault code bit scanner) to identify the next fault bit, then 0x79C7 (blink pattern generator) to set up the blink timing. If no faults and no active blink sequence, calls 0x7A16 (idle toggle timer) to produce a steady blink.

Phase 4 — Test mode (0x78C7–0x7927): Similar to Phase 3 but uses current fault registers (0x332E, 0x3330) instead of latched registers. Also includes an extended idle-check path: if no faults, checks engine conditions (0x3051 bit 2, elapsed timer 0x318C < 0x030D, engine state 0x3030 bits 2,3,7, intermittent flag 0x31EA bit 7) to determine whether to toggle the lamp or hold it steady.

Inputs

  • 0x35A4 bit 0: self-test mode active
  • 0x35A5: port 0x2000 shadow register
  • 0x3510: diagnostic mode control (0 = normal, non-zero = test)
  • 0x305C bit 2: clear-codes mode flag
  • 0x3051 bit 2: engine running indicator
  • 0x332E, 0x3330: current fault flags
  • 0x350A, 0x350C: latched fault flags
  • 0x3216: persistent fault counter (16-bit)
  • 0x318C: diagnostic elapsed timer (16-bit)
  • 0x3030 bits 2,3,7: engine condition flags
  • 0x31EA bit 7: intermittent fault detection ready

Outputs

  • 0x3190: MIL blink timer / reload counter
  • 0x3191: MIL blink phase timer (set by 0x79C7)
  • 0x3192: current fault code being blinked (set by 0x7928)
  • 0x318E: fault persistence counter (16-bit)
  • 0x35A5 bit 5: MIL lamp state (via port_2000_shadow_write 0x920B)
  • Port 0x2000 bit 5: physical MIL lamp output

Calls

  • 0x6346 (saturating_decrement): timer countdown
  • 0x634E (saturating_increment_y): persistence counter
  • 0x7928 (fault_code_bit_scanner): find next fault to blink
  • 0x79C7 (blink_pattern_generator): set up blink timing
  • 0x7A16 (idle_toggle_timer): idle-state lamp toggle
  • 0x920B (port_2000_shadow_write): update physical port
