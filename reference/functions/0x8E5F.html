Summary

Bidirectional ramp integrator for a boost-related accumulator (0x3228). Operates in two modes depending on 0x305B bit 0: increment mode (bit 0 clear) or decrement mode (bit 0 set). Each mode has a debounce counter at 0x3226/0x3227 that must reach a ROM threshold before the accumulator is adjusted. The accumulator is clamped to a ROM maximum (0x4A96) and minimum of zero.

On entry, loads an initial value from ROM 0x4A17 into 0x30FD, which boost_control_main later uses as a timing/condition flag.

Inputs

  • 0x305B bit 0: direction flag (0 = increment, 1 = decrement)
  • 0x3226 (decrement debounce counter)
  • 0x3227 (increment debounce counter)
  • 0x3228 (16-bit accumulator)

Outputs

  • 0x30FD: loaded from ROM 0x4A17 (only in increment path)
  • 0x3226/0x3227: debounce counters (one cleared, other incremented per call)
  • 0x3228: updated accumulator (clamped to 0..ROM[0x4A96])

Calls

  • 0x633D -- saturating_increment

RAM Reads

  • 0x305B, 0x3228

RAM Writes

  • 0x30FD, 0x3226, 0x3227, 0x3228

ROM References

  • 0x4A17 -- initial/timing value stored to 0x30FD
  • 0x4A90 -- increment debounce threshold
  • 0x4A91 -- decrement debounce threshold
  • 0x4A92 -- increment step size (16-bit)
  • 0x4A94 -- decrement step size (16-bit)
  • 0x4A96 -- maximum accumulator value (16-bit)
