Summary

Two-phase cranking/idle speed sensor check that validates the neutral/park position switch and idle contact signals against RPM. The first phase (0x6EFF–0x6F78) checks switch consistency at two RPM ranges: low RPM (4–20, cranking/idle) and high RPM (>=40, running). Each range has its own debounce counter and fault bit. The second phase (0x6F79–0x6FC6) handles diagnostic mode confirmation and cross-checks both port directions to detect intermittent faults, with a "both-sides-seen" latch pattern using 0x32B1.

Phase 1: RPM-Based Switch Check (0x6EFF–0x6F78)

  1. Precondition gates (0x6EFF–0x6F0C): Exit if 0x3330 bit 0 set (prior fault) or 0x332F bit 6 set (ignition fault)
  2. Engine running (0x6F0D): BBC 0x3051 bit 2 — engine must be running
  3. RPM load (0x6F12): LDX 0x3062 — load 16-bit RPM value
  4. RPM range routing:
    • RPM < 4: exit (too low, no valid data)
    • RPM 4–20 (0x6F21): Low-RPM path (cranking/idle)
      • Clear high-RPM counter 0x3189
      • Check vehicle speed 0x3071 == 0 (must be stationary)
      • Check port 0x2000 bit 2 (idle switch): if set, clear counter and fault
      • If idle switch NOT set: increment counter 0x3188, threshold 0x27 (39 counts)
      • On trip: SEB 0x3032 bit 4 (low-speed idle fault)
    • RPM >= 20 (0x6F49): High-RPM path (running)
      • Clear low-RPM counter 0x3188
      • RPM must be >= 40 (0x28) for test to proceed
      • Check port 0x2000 bit 2 (idle switch): if NOT set, clear counter and fault
      • If idle switch set at high RPM: increment counter 0x3189, threshold 0x62 (98 counts)
      • On trip: SEB 0x3032 bit 5 (high-speed idle fault)

Phase 2: Diagnostic Confirmation (0x6F79–0x6FC6)

  1. Normal mode (0x3510 == 0): Check 0x3032 bits 4+5 — if either fault set, promote to 0x3330 bit 1
  2. Diagnostic mode (0x3510 != 0):
    • Check 0x32B1 bit 5 (already in fault tracking) or 0x31CF bit 7 (pending fault from other test)
    • Cross-check pattern using 0x32B1 bits 5/6/7:
      • Port 0x2000 bit 2 = 0 → set 0x32B1 bit 6 ("off seen")
      • Port 0x2000 bit 2 = 1 → set 0x32B1 bit 7 ("on seen")
      • Both bits 6+7 set → SEB 0x31EA bit 7 (cross-check passed — intermittent fault confirmed)
    • If cross-check not complete: check timer 0x318C >= 0x030D before promoting to 0x3330 bit 1

Inputs

  • 0x3330 bit 0: prior VSS fault (inhibits test)
  • 0x332F bit 6: prior ignition fault (inhibits test)
  • 0x3051 bit 2: engine running
  • 0x3062: RPM (16-bit)
  • 0x3071: vehicle speed (must be 0 for low-RPM path)
  • 0x2000 bit 2: idle/closed-throttle switch (I/O port)
  • 0x3510: diagnostic mode control
  • 0x31CF bit 7: pending fault from other test
  • 0x318C: diagnostic elapsed timer (16-bit)

Outputs

  • 0x3032 bit 4: low-RPM idle switch fault
  • 0x3032 bit 5: high-RPM idle switch fault
  • 0x3188: low-RPM fault debounce counter (8-bit)
  • 0x3189: high-RPM fault debounce counter (8-bit)
  • 0x32B1 bits 5/6/7: diagnostic cross-check latch (both-sides-seen pattern)
  • 0x31EA bit 7: intermittent fault confirmed
  • 0x3330 bit 1: confirmed neutral/park switch fault
