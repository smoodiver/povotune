0xDBBB — Idle Dashpot Controller

Field Value
Address 0xDBBB
Size 75 bytes (0xDBBB–0xDC05)
Blocks 14
Type subroutine
Callers 1 — scheduler ISR at 0xAABE
Confidence confirmed

Summary

Manages an idle dashpot (deceleration air valve) timer that holds a duty cycle override for a countdown period after specific conditions are met. The function monitors port $2000 bit 1 (idle switch state) and uses a state latch in $322C bit 7 to detect transitions. When the idle switch releases (port bit 1 clears) under qualifying speed/RPM conditions, loads a timer value and returns an override duty value. Otherwise, counts down the existing timer and returns a default value.

Flow

  1. Port check: Tests $2000 bit 1:

    • If set: sets $322C bit 7 (idle switch active latch), skips to countdown path
    • If clear: tests $322C bit 7 for edge detection
      • If $322C bit 7 was set: clears it (falling edge detected), enters qualification
      • If already clear: skips to countdown path

  2. Qualification (falling edge only):

    • $3071 (vehicle speed) < ROM $42D2
    • $3062 (RPM) < ROM 16-bit $42D3 and RPM > ROM 16-bit $42D5 (RPM in window, with equality check on lower bound)
    • If all pass: loads $42D0 into timer $32C3, returns with A = ROM $42D1 (dashpot duty override)

  3. Countdown path: If $32C3 > 0, decrements it. While nonzero, returns A = ROM $42D1. When timer reaches 0, returns A = 0.

Inputs

  • Stack: caller's A register is preserved/restored via PHA/PLA

Outputs

  • A register: override duty value (ROM $42D1 while timer active, 0 when expired)
