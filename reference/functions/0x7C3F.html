0x7C3F — Self-Test Toggle Timer

Field Value
Address 0x7C3F
Size 28 bytes (0x7C3F–0x7C5A)
Blocks 5
Type subroutine
Callers 1 call site (self_test_entry chain)
Confidence confirmed

Summary

Small timer-gated toggle routine within the self-test subsystem. Checks whether self-test is active (0x35A4 bit 0). If not active, returns immediately. If active, increments a counter at 0x32AF via saturating increment, and when it reaches 20 (0x14), resets the counter to zero and toggles bit 7 of 0x35A4.

The toggle of bit 7 on 0x35A4 appears to be a periodic heartbeat or phase-flip signal. Since the self-test status register (0x35A4) is readable via the Subaru Select Monitor, toggling bit 7 at a known rate gives the diagnostic tool a way to verify the ECU is alive and cycling through its self-test loop.

Inputs

  • 0x35A4 (diag_self_test_status): bit 0 gates entry (self-test active flag)
  • 0x32AF (self_test_toggle_counter): current count

Outputs

  • 0x32AF (self_test_toggle_counter): incremented, or reset to 0 on overflow
  • 0x35A4 (diag_self_test_status): bit 7 toggled every 20 calls

Calls

  • 0x633D — saturating_increment: increments byte at X

RAM Reads

  • 0x35A4 (diag_self_test_status): bit 0 test at entry
  • 0x32AF (self_test_toggle_counter): read via saturating_increment

RAM Writes

  • 0x32AF (self_test_toggle_counter): incremented or cleared to 0
  • 0x35A4 (diag_self_test_status): bit 7 toggled (EOR #$80)
