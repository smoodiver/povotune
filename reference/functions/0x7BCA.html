0x7BCA — Self-Test Entry

Field Value
Address 0x7BCA
Size 117 bytes
Blocks 15
Type subroutine
Callers 1 call site
Confidence confirmed

Summary

Periodic self-test routine with multi-phase execution. Phase 1: checks preconditions (not in certain modes, vehicle speed >= 150), then activates self-test by setting 0x35A4 bit 0, calling RAM test (0x7C9D), and jumping to reset init. Phase 2: runs ROM checksum (0x7C5B), sets hardware_config. Phase 3: processes diagnostic results from 0x356F–0x3571 into scaled outputs at 0x3574–0x3578.

Inputs

  • 0x3051 (map_mode_control): bit 2 gates entry
  • 0x305A (system_control_flags): bit 0 gates entry
  • 0x3071 (vehicle_speed): must be >= 0x96 (150)
  • 0x35A4 (diag_self_test_status): phase tracking bits

Outputs

  • 0x35A4 (diag_self_test_status): bits 0,1,2 set during phases
  • 0x32C2: bit 0 set when continuing test
  • 0x3505 (hardware_config): set to 0x01 after checksum
  • 0x3574, 0x3576, 0x3578: diagnostic result outputs (scaled)

Calls

  • 0x633D — unknown: timer/counter comparison function
  • 0x7C9D — RAM test: memory integrity check
  • 0x7C5B — ROM checksum: ROM integrity verification
  • 0x65B5 — reset_init_continued: system restart (JMP, not JSR)
