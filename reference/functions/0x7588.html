0x7588 — Fault Threshold Evaluator

Field Value
Address 0x7588
Size ~396 bytes (0x7588–0x7713)
Blocks 71
Type subroutine
Callers 1 (health_check_dispatcher 0x6B14, slot 23)
Confidence confirmed

Summary

Multi-channel fault threshold evaluator that examines engine operating conditions against ROM-defined thresholds and accumulates per-channel debounce counters. Uses six independent debounce counters (0x3343–0x3348) to track fault conditions across different operating zones. When any counter reaches its trip threshold, sets status bits in 0x333B (bits 2, 3, 4) and promotes to confirmed fault via 0x3331 bit 5.

The function has three major phases:

Phase 1 — Gate checks (0x7588–0x759F): Checks three control registers (0x332E bit 4, 0x332F bit 4, 0x3331 bit 4). If any is set, clears all six counters and exits. Otherwise checks 0x3291 bit 3 — if clear, jumps to a "clear all" path.

Phase 2 — Counter evaluation (0x75B6–0x76FE): Six parallel evaluation channels, each gated by coolant temperature (0x3044) compared against ROM threshold pairs. When temperature exceeds the threshold AND additional conditions are met (load vs ROM table, speed vs threshold, turbo duty vs threshold), the corresponding counter is incremented via saturating_increment. Counters are cleared when conditions are not met.

  • Channel 1 (0x3343): Coolant > ROM[0x4830], load lookup via table 0x49C0 compared to turbo_duty_feedback (0x32AE)
  • Channel 2 (0x3344): Coolant > ROM[0x4832], load (0x3304) > ROM[0x4833]
  • Channel 3 (0x3345): Coolant > ROM[0x4835], load (0x3304) >= ROM[0x4836]
  • Channel 4 (0x3346): Coolant > ROM[0x4838], turbo_duty_feedback >= ROM[0x4839]; requires 0x3275 bit 2 SET
  • Channel 5 (0x3347): Coolant > ROM[0x483B], load lookup via table 0x49D0 compared to load (0x3304)
  • Channel 6 (0x3348): Speed (0x3279) lookup via table 0x5580 compared to turbo_duty_feedback (0x32AE)

Phase 3 — Threshold check and flag promotion (0x767B–0x7713): Compares each counter against its individual ROM trip threshold (0x4831, 0x4834, 0x4837, 0x483A, 0x483C, 0x483D). If any counter reaches its threshold, sets the corresponding bit in 0x333B. Final check: if 0x333B bits 2+3+4 (mask 0x1C) have any set, promotes to 0x3331 bit 5.

Inputs

  • 0x332E bit 4, 0x332F bit 4, 0x3331 bit 4: gate flags (any set = clear all)
  • 0x3291 bit 3: map control register (must be set for evaluation)
  • 0x333B bit 5: inhibit flag (blocks channel 1)
  • 0x3044 (coolant_temp): compared against per-channel ROM thresholds
  • 0x3304 (load_value): compared against per-channel ROM thresholds
  • 0x306F (airflow): lookup input for table interpolation
  • 0x3279 (turbo_speed_threshold): lookup input for channel 6
  • 0x32AE (turbo_duty_feedback): comparison value for channels 1, 4, 6
  • 0x3275 bit 2: turbo active flag (gates channel 4, inhibits channels 1-3)

Outputs

  • 0x3343–0x3348: six per-channel debounce counters
  • 0x333B bits 2, 3, 4: individual channel trip flags
  • 0x3331 bit 5: confirmed fault aggregate flag

Calls

  • 0x6000 (1d_table_interpolation): channels 1, 5
  • 0x601A (1d_table_interpolation_16): channel 1 (speed multiplier), channel 6
  • 0x633D (saturating_increment): all six channels

Channels 1, 2, 3, 5, 6 are active with real thresholds. Only channel 4 (turbo-active overboost) is disabled (0xFF). The coolant gates at 0x5E/0xBB correspond to approximately 50–80°C operating range.

  • 0x49C0: 1D lookup table for channel 1 load calculation
  • 0x49D0: 1D lookup table for channel 5 load comparison
  • 0x5580: 16-entry lookup table for channel 6 speed/duty calculation

DTC Output

  • 0x3331 bit 5 maps to DTC code 66 (6 long + 6 short blinks) via the lookup table at 0x79A7 (see 0x7928.md)
  • Code 66 = turbo duty threshold fault — the turbo system's operating parameters (duty cycle) are outside expected range for current load/speed/temperature conditions
  • This is distinct from individual solenoid wiring faults (codes 62–64, 67 via sub_73AF–sub_74C3) and boost solenoid feedback fault (code 24 via sub_71AA)
