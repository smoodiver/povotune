0x6C76 — TPS/Throttle Sensor Check

Summary

Two-part throttle position sensor diagnostic. Part 1 checks TPS raw value (0x324C) for out-of-range: if <= 0x0014 or >= 0x03CA (shorted or open circuit), increments debounce counter 0x317D. After 4 consecutive fault counts, sets 0x3032 bit 6 (TPS range fault). Part 2 checks TPS idle contact consistency: if coolant temp (0x30DC) <= 0x50 and engine run time (0x30B6) >= 0x06DE, computes the difference between expected idle TPS (0x3196) and actual (0x30DC). If the difference is < 5 (stuck near idle position), increments counter 0x317E; after 4 counts sets 0x3032 bit 7 (TPS idle stuck fault). Finally, if either fault bit (0x3032 bits 6 or 7) is set, sets 0x332F bit 0 (combined TPS/VSS fault flag); otherwise clears it.

Inputs

  • 0x324C: TPS raw value (16-bit)
  • 0x30DC: coolant temperature or TPS-related sensor reading
  • 0x30B6: engine run timer (16-bit)
  • 0x3196: expected idle TPS reference / AD temp lookup result

Outputs

  • 0x317D: TPS range fault debounce counter
  • 0x317E: TPS idle stuck fault debounce counter
  • 0x3032 bit 6: TPS range fault flag
  • 0x3032 bit 7: TPS idle stuck fault flag
  • 0x332F bit 0: combined TPS/VSS fault condition

Calls

  • 0x633D: saturating_increment (counter 0x317D for range check)
  • 0x633D: saturating_increment (counter 0x317E for idle stuck check)

RAM Reads

  • 0x324C, 0x30DC, 0x30B6, 0x3196, 0x3032

RAM Writes

  • 0x317D, 0x317E, 0x3032, 0x332F
