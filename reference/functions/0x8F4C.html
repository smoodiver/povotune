Summary

Computes the maximum boost duty cycle limit from coolant temperature lookup tables with conditional offsets and a low-speed bypass path. Two computation paths:

  1. Low-speed path (0x305D bit 5 clear, RPM below threshold, speed below threshold): Uses the result from the low-speed table (0x4BA0) directly.
  2. High-speed path (all other cases): Uses the high-speed table (0x4BC0) result.

Both paths look up coolant_temp via graph_lookup_2d, then add the idle duty offset via table_lookup_result_add_offset (0x90AF). The low-speed path additionally applies up to three conditional offsets:

  • AT mode offset (0x4BEA) if 0x3210 bit 5 is set
  • Decel/idle offset (0x4BEC) if 0x305D bit 5 is set AND 0x305B bit 0 is clear
  • Port control offset (0x4BEE) if 0x3275 bits 0 or 3 are set

Result is clamped to 0xFFFF on overflow and stored to 0x3104 (low-speed) or 0x3106 (final output).

Inputs

  • 0x3093 (coolant_temp): temperature for table lookups
  • 0x3062 (rpm): engine speed
  • 0x3071 (vehicle_speed): vehicle speed
  • 0x3210 bit 5: AT/MT mode flag
  • 0x305B bit 0: condition flag
  • 0x305D bit 5: operating mode flag
  • 0x3275 bits 0, 3: port control mirror bits

Outputs

  • 0x3104: low-speed max duty result (intermediate, 16-bit)
  • 0x3106: final max duty limit (16-bit)

Calls

  • 0x6000 -- graph_lookup_2d (2 calls)
  • 0x90AF -- table_lookup_result_add_offset (2 calls)
