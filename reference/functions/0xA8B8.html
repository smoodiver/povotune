0xA8B8 — Knock Correction Clamp

Field Value
Address 0xA8B8
Size 143 bytes (0xA8B8–0xA946)
Blocks 19
Type subroutine
Callers 1 — knock_learning_update (0xA826 at 0xA8AB)
Confidence confirmed

Summary

Clamps and scales the knock correction value in B to safe limits. Implements a two-stage limiting:

  1. Positive correction (B >= 0): Computes a dynamic maximum based on the knock multiplier and advance timing, then clamps B to the lesser of the computed maximum or the ROM hard maximum. The formula is: max = min(B, (0x10 - knock_multiplier) * advance_timing >> 4), clamped to ROM limit $4DA0.
  2. Negative correction (B < 0): Simply clamps to the ROM minimum $4DA1.

After clamping, a secondary check evaluates whether the correction should be canceled entirely: if the advance timing ($31C7) exceeds a ROM threshold ($4DA2) and certain zone conditions are met, it computes a scaled comparison and may clear the knock zone flags, ending the knock control cycle.

Inputs

  • B — unclamped knock correction value (signed byte)
  • $3503 (knock_multiplier) — current knock multiplier (0–16)
  • $31C7 (advance_timing) — current ignition advance

Outputs

  • B — clamped knock correction value
  • May clear $3033 bit 6 and $3034 bit 2 (canceling knock control cycle)
