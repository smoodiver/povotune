0x63E0 — 16-Bit Table Interpolation

Field Value
Address 0x63E0
Size 81 bytes (0x63E0–0x6430)
Blocks 8
Type subroutine
Callers 2 call sites — 0xACF1 (IRQ handler, ADC input lookup), 0xB6FF (IRQ handler, speed/duty lookup)
Confidence confirmed

Summary

16-bit 1D table interpolation subroutine. Takes a 16-bit input value in A and a ROM table base pointer in 0x300A, splits the input into an integer index (upper bits) and a 4-bit fraction (lower bits), then linearly interpolates between the two bracketing 16-bit table entries. Returns the interpolated 16-bit result in A. Operates in 16-bit accumulator mode (M=0) throughout.

Algorithm

  1. Fraction extraction (0x63E0–0x63F4): Clears 0x3008, then right-shifts A four times while rotating carry into 0x3008. Result: A = input >> 4 (integer table index, 0–62 max), 0x3008 = fractional part (4-bit, left-justified in byte).
  2. Range check (0x63F5–0x63F8): If index >= 63 (0x3F), branch to clamp path at 0x642B which returns the last table entry directly.
  3. Address computation (0x63FA–0x63FE): ASL A (multiply index by 2 for 16-bit entries), add table base pointer (0x300A). TAX to use as index register.
  4. Interpolation (0x63FF–0x642F):
    • Load upper neighbor: LDA $02,X (table[index+1])
    • Compute difference: SEC; SBC $00,X (upper - lower)
    • If zero: return lower value directly
    • If positive: multiply difference by fraction via MPY, add to lower value
    • If negative: negate difference, multiply by fraction via MPY, subtract from lower value
  5. Return (0x6430): RTS with interpolated result in A.

Inputs

  • A register: 16-bit input value (upper bits = index, lower 4 bits = fraction)
  • 0x300A: 16-bit pointer to ROM table base (e.g., 0x4080, 0x5500)

Outputs

  • A register: 16-bit interpolated table value

RAM Reads

  • 0x300A (table_base_ptr) — pointer to ROM lookup table

RAM Writes

  • 0x3008 (interp_fraction) — 4-bit fractional part extracted from input, used as interpolation weight

ROM References

  • Indirect via 0x300A: ROM table of 16-bit entries (up to 64 entries, accessed as word pairs via X-indexed addressing at $00,X and $02,X)
  • Known table pointers: 0x4080 (from 0xACF1 caller), 0x5500 (from 0xB6FF caller)
