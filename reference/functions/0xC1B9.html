0xC1B9 — Turbo Transition Gain Calculator

Size 450 bytes (0xC1B9–0xC39A)

Type subroutine

Confidence confirmed

Summary

Computes four gain/correction coefficients used by the turbo transition controller (0xBDFF) for sequential turbo switchover dynamics. The function performs ROM table lookups based on RPM and produces asymmetric up/down ramp rates for turbo duty transitions.

The function has four phases:

  1. RPM scaling (0xC1B9–0xC1D9): Reads the raw RPM value at $301A, right-shifts by 5 to produce a coarse RPM index, clamps each byte independently to 0xFF. Stores the high portion to $3009 and low portion to $3008.

  2. Four table lookups (0xC1E4–0xC21F): Performs four calls to the 1D table interpolation routine (JSR $6000) using different ROM tables:

    • $3008 through ROM table at $45B0 -> $331B (ramp gain A, floor clamped to 1)
    • $3008 through ROM table at $45C0 -> $331C (ramp gain B, floor clamped to 1)
    • $3009 through ROM table at $45A0 -> $331A (base correction factor)
    • $3009 through ROM table at $45D0 (or $45E0 if $305C bit 5 set) -> $331D (directional bias)

  3. Speed-based gain selection (0xC222–0xC265): Selects a speed-dependent multiplier pair ($331E, $331F) using vehicle speed ($3071) and boost pressure ($30DC) compared against ROM thresholds at $45F0–$45F7. The $305C bit 5 selects between two ROM threshold sets, providing mode-dependent behavior.

  4. Asymmetric ramp computation (0xC268–0xC37A): Computes four output coefficients using signed arithmetic around a center point of 0x80:

    • $3316 (up-ramp duty subtract rate): (|$331D - 0x80| +/- $331A) * $331F, using MPY multiply
    • $3317 (down-ramp duty add rate): inverse computation of the same inputs
    • $3318 (up-ramp scaled gain): ($331A * $331F) >> 3, multiplied by $331B, with rounding
    • $3319 (down-ramp scaled gain): ($331A * $331F) >> 3, multiplied by $331C, with rounding

The signed arithmetic treats $331D as a signed offset around 0x80 (neutral). Values below 0x80 add to the base correction on one path and subtract on the other, creating intentional asymmetry between turbo ramp-up and ramp-down rates.

Inputs

  • $301A — raw RPM timer value (16-bit)
  • $3071 — vehicle_speed
  • $30DC — boost pressure reading
  • $305C — mode_select_flags (bit 5 selects ROM table variant)

Outputs

  • $3008 — scratch: RPM index (low, 8-bit clamped)
  • $3009 — scratch: RPM index (high, 8-bit clamped)
  • $331A — base correction factor (from RPM lookup)
  • $331B — ramp gain A (from RPM lookup, min 1)
  • $331C — ramp gain B (from RPM lookup, min 1)
  • $331D — directional bias (from RPM lookup, mode-dependent)
  • $331E — speed-based multiplier A
  • $331F — speed-based multiplier B
  • $3316 — up-ramp duty subtract rate (used by 0xBDFF above-target path)
  • $3317 — down-ramp duty add rate (used by 0xBDFF below-target path)
  • $3318 — up-ramp scaled gain (used by 0xBDFF above-target path)
  • $3319 — down-ramp scaled gain (used by 0xBDFF below-target path)
