0x71AA — Boost Solenoid Fault Check

Summary

Monitors the turbo boost control solenoid for faults. Loads X with pointer 0x318A (a two-byte debounce counter workspace). When diagnostic preconditions are not met (0x3330 bit 5 clear, or 0x305E bit 8 clear), clears the counters and exits. When preconditions are met, checks 0x3031 bit 6 (closed-loop boost active) and 0x305E bit 0 (solenoid drive signal). If the solenoid is commanded on, reads 0x30DF (boost feedback/duty) and compares against 0x40; if above threshold, increments the counter via saturating_increment. When the counter reaches 0x27 (39), sets 0x31F8 bit 5 (boost solenoid fault). If feedback is below 0x08, clears the counter. A second path handles the solenoid-off case. At exit, checks 0x31F8 combined bits (mask 0x38) — if bit 5 is set along with bits 3-4, sets 0x332F bit 3 and calls 0x7A35 (health check output classifier); otherwise clears 0x332F bit 3 and zeroes 0x3211.

Inputs

  • 0x3330 (diag_fault_flags_B): bit 5 must be set (precondition gate)
  • 0x305E (engine_state_flags_e): bit 8 must be set
  • 0x3031: bit 6 — closed-loop boost mode indicator
  • 0x305E: bit 0 — solenoid drive state
  • 0x30DF: boost solenoid feedback/duty reading

Outputs

  • 0x31F8 (o2_sensor_status): bit 5 set/cleared (boost solenoid fault)
  • 0x332F (temp_override_flags): bit 3 set/cleared
  • 0x3211 (health_check_output): zeroed on fault clear
  • 0x318A, 0x318B: debounce counter pair

Calls

  • 0x633D — saturating_increment (debounce counter)
  • 0x7A35 — health check output classifier
