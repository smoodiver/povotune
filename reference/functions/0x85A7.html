0x85A7 — Knock Position Tracker

Field Value
Address 0x85A7
Size 52 bytes (0x85A7–0x85DA)
Blocks 12
Type subroutine
Callers 2 call sites (ignition_timing at 0x839E and 0x83C9)
Confidence confirmed

Summary

Tracks a position (0–7) through a ROM breakpoint table based on a zone input value, implementing asymmetric hysteresis. Used by ignition_timing to compute the RPM and load axis positions into the 8x8 knock learned values table at 0x3630. Upward movement is immediate (position advances as soon as the zone reaches the next breakpoint), while downward movement is delayed by a configurable step size — the zone must drop by step_size units below the current breakpoint before the position decreases. This prevents oscillation at boundaries while allowing fast response to increasing RPM/load.

Inputs

  • A register: current zone value (RPM zone from 0x31BF or load zone from 0x31BE)
  • B register: current axis position (0–7, from 0x323A or 0x323C)
  • X register: pointer to current breakpoint in ROM table (table_base + position)
  • Y register: pointer to step size constant in ROM (0x4D89 or 0x4D8A)
  • 0x300C: lower bound pointer (start of breakpoint table — 0x4700 or 0x4708)
  • 0x300E: upper bound pointer (end of breakpoint table — 0x4708 or 0x4710)

Outputs

  • A register: new axis position (0–7, 16-bit in CLM mode; high byte always 0)

RAM Reads

  • 0x300C (tracker_lower_bound)
  • 0x300E (tracker_upper_bound)
