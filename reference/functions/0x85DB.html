0x85DB — Knock Retard Calculator

Field Value
Address 0x85DB
Size 255 bytes (0x85DB–0x86D9)
Blocks 30
Type subroutine
Callers 1 call site (ignition_output_controller at 0x81AC)
Confidence confirmed

Summary

Detects knock conditions via a state machine, accumulates a retard amount over time when knock persists, scales it by engine temperature and current timing angle, and produces a knock-proportional reduction value (0x330F) that is subtracted from the dwell start angle. The function also performs a temperature-dependent knock sensitivity calculation and manages the knock event state through control register 0x330A.

Inputs

  • 0x306E: knock sensor input value
  • 0x3093: coolant temperature (drives knock sensitivity)
  • 0x3092: engine state / temperature value
  • 0x30DC: reference value (subtracted from 0x3092 for delta)
  • 0x3044: comparison value (used in knock event detection)
  • 0x3046: comparison value (subtracted from 0x3044)
  • 0x3023 bit 0: control flag (gates knock processing)
  • 0x31BB: final ignition timing angle (scales the output)
  • 0x330A: knock state register (bits 0–2 read and written)
  • 0x330B: accumulated knock retard amount
  • 0x3310, 0x3311: knock event counters

Outputs

  • 0x330F: knock-proportional dwell reduction — subtracted from dwell angle in ignition_output_controller
  • IO 0x0228: knock reduction mirror (diagnostic output)
  • 0x330B: updated knock retard amount (accumulated over time)
  • IO 0x0226: knock retard amount mirror
  • 0x330A: updated knock state flags
  • 0x330C: temperature-dependent knock sensitivity factor
  • 0x330D: engine-state knock scaling factor
  • 0x330E: intermediate knock product
  • 0x3310, 0x3311: updated knock event counters

Calls

  • 0x6000 — graph_lookup_2d: coolant temperature → knock sensitivity (table 0x4C50)
