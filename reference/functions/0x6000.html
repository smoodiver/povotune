0x6000 — 1D Table Interpolation (8-entry)

Field Value
Address 0x6000 (primary), 0x600D (8-entry alternate), 0x601A (16-entry mode)
Size 13 bytes (0x6000–0x600C)
Blocks 4 (entry stub only — shared interpolation body attributed to sub_601A by flood-fill)
Type subroutine (three entry points into shared interpolation body)
Callers 62 via 0x6000 (70 total across all entry points: +1 via 0x600D, +7 via 0x601A)
Confidence confirmed

Summary

Core 1D linear interpolation utility. Takes a scaled input value in A and a ROM table base pointer in X, clamps the input to prevent overflow, divides it into an integer table index and fractional part, then performs linear interpolation between adjacent table entries. Three entry points select different table sizes and shift counts.

Entry Points

0x6000 — 8-entry table, divide-by-32 (62 callers)

  • Clamps A to 0xF0 max
  • 5 right-shifts: input / 32 = index (0–7), remainder = fraction
  • Max valid input 0xF0 yields index 7 (last pair: entry 7 and 8 would overflow, but 0xF0/0x20 = 7.5)

0x600D — 8-entry table, divide-by-8 (1 caller)

  • Clamps A to 0xF8 max
  • 3 right-shifts: input / 8 = index (0–7)
  • Finer resolution per table step

0x601A — 16-entry table, divide-by-128 (7 callers)

  • Clamps A to 0xE0 max
  • 7 right-shifts (2 extra + 5 shared): input / 128 = index (0–15... actually 0–7 with 0xE0/0x80 ≈ 1.75)
  • Wait — 0xE0 >> 7 = 1, so this gives a very coarse 2-entry interpolation. More likely: 0xE0 clamp with 7 shifts allows wider input range.

Algorithm Detail

  1. Clamp: CMP/BCC/BEQ pattern caps A at entry-specific maximum
  2. Split: B = 0, then LSR A / ROR B chain shifts the 16-bit A:B right. Integer part stays in A (table index), fractional part goes to B (interpolation weight, 0x00–0xFF)
  3. Address calc: CLM (binary mode), mask A to 8-bit, add X (table base) to get absolute ROM address in X
  4. Interpolate:
    • If B == 0: return table[index] directly (no fractional part)
    • Load delta = table[index+1] - table[index]
    • If delta < 0: negate, multiply by B (MPY), subtract from table[index+1]
    • If delta >= 0: multiply by B (MPY), add to table[index]
  5. Return: Interpolated result in A

Inputs

  • A — scaled input value (e.g., sensor reading, RPM derivative). Clamped to entry-point-specific maximum
  • X — ROM table base address (pointer to first byte of lookup table)

Outputs

  • A — interpolated 8-bit result from the table

RAM Reads

  • 0x3060 (scratch) — used as temporary for X storage and MPY operand

RAM Writes

  • 0x3060 (scratch) — temporarily stores X register value and multiplication operand

ROM References

  • None (table addresses are passed in X by callers)
