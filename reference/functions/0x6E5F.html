0x6E5F — Ignition Signal Detect

Field Value
Address 0x6E5F
Size 32 bytes (0x6E5F–0x6E7E)
Blocks 6
Type subroutine
Callers 1 call site (health_check_dispatcher 0x6B14, slot 9)
Confidence confirmed

Summary

Monitors for loss of ignition signal by checking bit 5 of 0x3253 (ignition signal present flag). When the ignition signal is present, the fault counter is cleared and the fault flag is removed. When absent, a counter at 0x3139 is incremented each cycle; when it reaches 0x14 (20), the ignition signal loss fault flag is set.

Disassembly

6E5F  2C 53 32 20 0C BBS $3253,#$20,$6E70   ; If ignition signal present -> clear path
6E64  EE 39 31       INC $3139               ; Increment loss counter
6E67  AD 39 31       LDA $3139               ; Load counter value
6E6A  C9 14          CMP #$14               ; Trip threshold = 20
6E6C  B0 0C          BCS $6E7A               ; If >= 20 -> set fault
6E6E  80 0E          BRA $6E7E               ; Else -> return
6E70  1C 2F 33 02    CLB $332F,#$02         ; Clear ignition loss fault flag
6E74  9C 39 31 00    LDM $3139,#$00         ; Clear loss counter
6E78  80 04          BRA $6E7E               ; -> return
6E7A  0C 2F 33 02    SEB $332F,#$02         ; Set ignition loss fault: 0x332F bit 1
6E7E  60             RTS

Inputs

  • 0x3253 bit 5: ignition signal present flag
  • 0x3139: ignition signal loss counter

Outputs

  • 0x3139: updated loss counter (incremented or cleared)
  • 0x332F bit 1: ignition signal loss fault flag

RAM Reads

Address Name Use
0x3253 ignition_signal_flags Bit 5: ignition signal present
0x3139 ign_signal_loss_counter Running count of cycles without signal

RAM Writes

Address Name Use
0x3139 ign_signal_loss_counter Incremented on no signal, cleared on signal present
0x332F temp_override_flags Bit 1: crank position signal loss / ignition signal fault

None (threshold 0x14 is an immediate operand).
