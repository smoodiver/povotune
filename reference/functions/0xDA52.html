0xDA52 — RPM Acceleration Filter

Field Value
Address 0xDA52
Size 156 bytes (0xDA52–0xDAED)
Blocks 26
Type subroutine
Callers 1 — scheduler P2 (0xAAD1 at 0xAB30)
Confidence confirmed

Summary

Computes a filtered signed RPM acceleration value used by the ignition timing retard controller (0xD90A). Processes the previous signed accumulator ($326C), applies a signed arithmetic shift right by 4 with rounding, adds current RPM ($3064), subtracts previous RPM snapshot ($3266), scales the delta by a gain of 15 (MPY #$0F), integrates with the previous accumulator ($326A) using signed saturation, and stores the result. The final output is doubled (ASL) with saturation and stored to $3008, from which $3009 (high byte) is copied to $326E as the 8-bit acceleration output.

Called every P2 scheduler tick (every ~4th ISR tick), providing a smooth signed RPM rate-of-change estimate.

Algorithm

  1. Signed shift right by 4: Load $326C (16-bit signed). If positive, AND with 0xFFF8 then CLC; if negative, OR with 0x0007 then SEC. Rotate right 4 times (arithmetic shift preserving sign via carry).
  2. Delta computation: Add $3064 (current RPM full precision), subtract $3266 (previous RPM snapshot). Result is the instantaneous RPM change + filtered history.
  3. Gain scaling: If delta >= 0, multiply by 15 (MPY #$0F), clamp to 0x7FFF. If delta < 0, negate, multiply by 15, clamp to 0x8000 (negate back).
  4. Integration: Add result to $326A (previous accumulator, signed). Saturate at +0x7FFF or -0x8000 depending on signs.
  5. Store: Result to $326C. Double via ASL with signed saturation to $3008.
  6. Output capture: Switch to 8-bit mode (SEM), copy $3009 (high byte of $3008) to $326E. Snapshot $3064 to $3266 for next iteration.
