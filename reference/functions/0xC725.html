0xC725 — Speed Control

Summary

Comprehensive speed control and limiter state machine. Implements multiple hysteresis comparators that compare vehicle speed (0x3071) and airflow/load (0x306F) against ROM threshold tables, setting and clearing control flags in 0x302A (speed control register) and 0x31E0 (speed limit flags). Uses three parallel 2D lookup paths (ROM tables at 0x4E50, 0x4E70, 0x4E90) selected by speed control state bits, each performing a table lookup via JSR $6000 with optional correction from secondary tables (0x46E2–0x46EA). Downstream subsystems (fuel, ignition) read these flags to implement speed limiting and power reduction.

Inputs

  • 0x3071: current vehicle speed — primary input
  • 0x306F: airflow/load value
  • 0x3093: coolant temperature — processed AD value
  • 0x302A: speed control register — bits read and written
  • 0x305A: system control flags
  • 0x305C: system control flags
  • 0x3092: engine state value
  • 0x301A: AFM storage value
  • 0x31E0: speed limit flags
  • 0x314E: limiter flags
  • 0x32E0, 0x3314: limiter input values
  • 0x3510, 0x332F, 0x3331, 0x333B: additional control/status registers

Outputs

  • 0x302A bits 0–5, 7: speed control state flags
    • bit 0 (0x01): speed below minimum / in low-speed mode
    • bit 1 (0x02): speed limit lookup active
    • bit 2 (0x04): speed exceeded upper threshold (hysteresis-controlled)
    • bit 3 (0x08): speed limit enforced
    • bit 4 (0x10): speed limit counter active
    • bit 5 (0x20): AFM/load threshold exceeded
    • bit 7 (0x80): high-speed AFM limit active
  • 0x31E0 bits 0–1: speed level flags (two-tier speed limiter with hysteresis)
    • bit 0 (0x01): speed level 1 exceeded
    • bit 1 (0x02): speed level 2 exceeded
  • 0x314E bits 2–5: limiter flags (RPM/load thresholds)
  • 0x30D3: speed limit output — set to 0xFF when limiting, 0x00 when not
  • 0x320D: speed limit counter

Calls

  • 0x6000 — graph_lookup_2d: 2D table lookup (called 3x — one per speed control path)
