0x6378 — Turbo Duty Map Lookup

Field Value
Address 0x6378
Size 104 bytes (0x6378–0x63DF)
Blocks 9
Type subroutine
Callers 2 — from 0xCE03 (Call 1) and 0xD178 (Call 2) in twin_turbo_duty_controller
Confidence confirmed

Summary

8×8 nearest-neighbor table lookup (no interpolation). Searches two 8-entry axis tables to find row and column indices, then reads a byte from the map at base + row×8 + col. Falls through to sub_63D4 for the final indexed read. Used exclusively by twin_turbo_duty_controller (0xCC9F) for turbo duty cycle target lookups.

Algorithm

  1. Column search (0x6378–0x6394): Load B=7 (counter), X=$308C (axis B ptr), A=$3091 (input B). Loop: compare A with axis[X+1]; if A < axis value, break; else advance. Column index = 7 - remaining_count. Push column index.
  2. Row search (0x6395–0x63C0): Load X=$308A (axis A ptr), B=$308E, A=$308F. Right-shift A:B pair by 4 bits (divide by 16, saturating B at 0xFF if A nonzero). Loop: compare B with axis[X+1]. Row index = 7 - remaining_count. Compute row_offset = row_index × 8.
  3. Index and read (0x63C0–0x63DF): linear_index = column + row_offset. Falls through to sub_63D4: A = ROM[$3088 + linear_index].

Inputs

  • 0x3088: map data base pointer (e.g., 0x4790 or 0x4940)
  • 0x308A: axis A pointer (searched with processed RPM)
  • 0x308C: axis B pointer (searched with $3091)
  • 0x308E: RPM value (16-bit, from $3062) — right-shifted 4 bits for axis A search
  • 0x308F: high byte of RPM (for shift chain)
  • 0x3091: throttle/load input (from $3044)

Outputs

  • A register: map value at the looked-up cell (8-bit)

Key Difference from 0x6080

Unlike the main map lookup engine (0x6080) which performs bilinear interpolation, this function uses nearest-neighbor lookup — it finds the closest axis bin and reads the cell directly. This makes it faster but coarser, suitable for the proportional control loop in the turbo duty controller where the step-adjustment provides smoothing.
