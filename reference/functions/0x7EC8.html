0x7EC8 — Fuel Map Lookup

NOTE: ROM[0x4039] selects between two fuel calculation strategies — likely a turbo/NA hardware variant switch. Path 1 (0x4039 == 0, turbo): full 16x16 fuel map indexed by RPM and load. Path 2 (0x4039 != 0, NA): simplified derivative-based calculation from AFM rate-of-change ((afm_delta >> 2) + 0x40). The historical analysis confirms this is a fuel map: FHI (factory) values are larger than tuned values at 0x4120, consistent with a tuner leaning out the mixture for performance.

Summary

Looks up the fuel duty cycle value from a 16x16 map indexed by RPM and engine load, or alternatively computes it directly from a RAM variable when ROM config byte 0x4039 is non-zero. The result is stored at 0x374A and mirrored to IO register 0x012E. A secondary output derives a shifted fuel value at 0x3122 from 0x3749, and the map lookup table index is copied from 0x3085 to 0x31EC.

Inputs

  • ROM[0x4039]: config byte — selects between map lookup (zero) and direct calculation (non-zero)
  • 0x3062: current RPM — drives map lookup axis
  • 0x313C: engine load value, 16-bit — drives map lookup axis
  • 0x32DE: direct calculation input (used only when ROM[0x4039] != 0)
  • 0x3749: fuel map low byte, 16-bit
  • 0x3085: map lookup table index

Outputs

  • 0x374A: fuel duty cycle result — also written to IO 0x012E
  • IO 0x012E: fuel duty cycle mirror (IO output)
  • 0x3122: shifted fuel value — high byte of 0x3749, right-shifted 2 bits
  • 0x31EC: table index copy from 0x3085

Calls

  • 0x6080 — map_lookup_16x16: bilinear interpolation over fuel/boost table (historical: confirmed)
