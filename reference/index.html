Getting Started

When you first open PovoTune, you have three options:

  • Load Project JSON — Open a previously saved .json project file
  • Analyze Binary — Load a raw .bin firmware dump and run automated analysis
  • Factory BD5A ROM — Load the pre-analysed BD5A factory ROM project instantly

You can also drag and drop a .json or .bin file directly onto the landing page.

Toolbar

The toolbar along the top of the screen provides:

  • View toggle buttons (ASM, Hex, Graph, Map) — Click to show or hide each view. Active views are highlighted. Any combination can be open simultaneously.
  • Menu (hamburger icon, top-right):
    • Load Project — Load a different project file
    • Download Project — Save your current project as a .json file
    • Project Settings — Configure memory regions and project metadata
    • Help — Open this page
    • Reset Session — Clear all saved data and start fresh

Memory Map Strip

The thin horizontal bar below the toolbar shows the full 64 KB address space (0x0000–0xFFFF), colour-coded by memory region.

ClickJump both ASM and Hex views to that address
HoverSee region name and address range

Left-Hand Navigation

The sidebar on the left has five collapsible sections. Click any header to collapse or expand it.

Functions

All detected firmware functions, sorted by address.

ClickNavigate ASM, Hex, and Graph to that function
Double-click nameEdit the function name inline (Enter to confirm, Escape to cancel)

The coloured badge on each row indicates analysis confidence level.

IO Registers

Memory-mapped I/O addresses discovered during analysis.

ClickNavigate Hex view to that address
Double-click nameEdit the label inline

RAM

RAM memory locations with their labels.

ClickNavigate Hex view to that address
Double-click nameEdit the label inline

Tables (Data Definitions)

All declared ROM data tables.

ClickNavigate Hex view to the table start address

ASM View

Disassembled firmware code displayed in a virtual-scrolled table with columns: Address, Bytes, Mnemonic, Operand, and Notes.

Navigation & Selection

Click instructionSelect it and show details in the Instruction Detail panel
Click branch targetBlue link in the operand column — jump ASM and Hex to that address
Click table refGold underlined link — jump Hex view to the referenced table
Click function headerBlue bar — select and focus that function

Visual Elements

  • Function headers — Blue bars showing function name, address, and size
  • Block boundaries — Dashed red lines separating basic blocks
  • Notes column — Supports inline annotation editing (click to edit, semicolon prefix)

Instruction Detail Panel

When an instruction is selected, the detail panel (bottom-right) shows:

  • Mnemonic and addressing mode
  • Byte length and raw bytes
  • M flag (memory width: 8-bit or 16-bit) and X flag (index register width)

Hex View

Interactive hex memory browser showing 16 bytes per row, colour-coded by object type.

Toolbar

ADDR + EnterType a hex address (4120, 0x4120, or $4120) to jump there
FIND + EnterSearch for byte patterns (e.g. A9 00). Bare hex searches bytes in order; prefixed hex (0x4120) searches little-endian
Shift+EnterFind previous match
Arrow buttonsNavigate between search matches

Selection

Shift+ClickStart or extend a byte range selection
Shift+ArrowExtend selection (Left/Right = 1 byte, Up/Down = 16 bytes)
EnterOpen context menu for current selection
EscapeClear selection, cancel edit, or dismiss search highlights

Right-Click Context Menu

On a Function

  • Navigate to ASM — Jump the disassembly view to this function
  • Rename — Edit the function name
  • Re-analyse — Re-run analysis on this function
  • Delete — Remove the function entry point

On a Data Definition (Table)

  • Rename — Edit the table name
  • Edit Range — Resize the table using arrow keys, Enter to confirm
  • Edit Type — Change element type (u8, u16, u32, s8, s16, s32)
  • Colourise / Uncolourise — Toggle heatmap visualisation across element values
  • Delete — Remove the data definition

On a Byte Selection

  • Define Table — Create a new data definition from the selection
  • Define Function — Create a new function entry point and run analysis

Always Available

  • Copy Address — Copy as 0xABCD
  • Copy Bytes — Copy selected bytes as hex (e.g. 41 20 30 44)

Colour Coding

  • Blue — Functions (decoded instructions)
  • Orange — Data definitions / tables (heatmap gradient when colourised)
  • Grey — Memory locations
  • Red — Interrupt vectors
  • Dark — Unmapped / unknown regions

Graph View

Interactive force-directed graph showing the selected node and its immediate neighbours (ego-centric view).

Interactions

Click nodeNavigate ASM, Hex, and sidebar to that function or table
Double-click nodeExpand that node's neighbours — it becomes the new centre
Drag nodeReposition it on the canvas (position is remembered)
Mouse wheelZoom in and out

Node Types

  • Rounded rectangle — Function
  • Rounded rectangle + "TBL" badge — Data definition / table
  • Circle + "RAM" badge — Memory location
  • Diamond + "VEC" badge — Interrupt vector

Edge Types

  • Solid blue — CALLS (function call)
  • Green dashed — READS_FROM / WRITES_TO (memory access)
  • Grey dashed — MAPS_TO (ROM-to-RAM table pointer)
  • Black solid — ENTRY_POINT (vector to function)

Edge labels show the relationship type and count, e.g. CALLS (3).

Map View

A 256×256 pixel grid on the right-hand side visualising the entire 64 KB address space. Each pixel represents one byte, coloured by content type.

Interactions

Click cellJump the Hex view to that address
HoverSee address, region, and object information

Teal cells indicate data definitions. Decoded instruction bytes use region-specific colours. The current cursor position is shown as a thin accent line.

Refs Panel

Shows cross-references to and from the selected address. Opens when you click a reference count in a hover tooltip.

Click ref rowNavigate to that address

References are colour-coded: calls (blue), branches (purple), memory access (green), maps (orange).

Saving Your Work

Your project is automatically saved to IndexedDB in your browser after every edit. To create a portable backup:

  • Use Menu → Download Project to export a .json file
  • All annotations, renamed functions, data definitions, and corrections are included
  • Reload the file later with Load Project JSON on any machine
