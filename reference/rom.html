ROM Tables — BD5A

ROM data tables and constants reference for the BD5A Subaru Legacy RS ECU ROM. One entry per table/constant, grouped by data type.

Source key: bd5a = derived from BD5A firmware function analysis. historical = from original manual reverse engineering (historical-analysis.md).

Axes

0x4100 — RPM Axis (Primary)

Field Value
Address 0x4100
Size 16 bytes
Format 16x1 unsigned 8-bit breakpoint table
Scaling value x 12.5 = RPM (historical; see notes)
Duplicate 0x44D0 (identical contents, used by ignition)
Source historical

Contents:

10 18 20 28 30 38 40 48 50 58 60 68 70 78 80 88
(16, 24, 32, 40, 48, 56, 64, 72, 80, 88, 96, 104, 112, 120, 128, 136)

Evenly spaced with step 8 (100 RPM at x12.5 scaling).

Used by:

  • sub_7EC8 (fuel_map_lookup) at 0x7EE8 — stored to 0x3074 as RPM axis for fuel table 0x4120
  • sub_CB52 (3-map selection) at 0xCB6F — stored to 0x3074 as RPM axis for maps 0x5200/0x5300/0x5400

Duplicate 0x44D0 used by:

  • sub_82D6 (ignition timing) at 0x82DC and 0x830D — stored to 0x3074 as RPM axis for base timing map 0x5100 and advance timing map 0x5000

RPM source: All three functions load the current RPM from 0x3062 → 0x307A. The value at 0x3062 is the raw timer period right-shifted twice (LSR; LSR at 0xBDA0–0xBDA1 in sub_BD51), making it inversely proportional to RPM. The axis breakpoints are matched against this value by the map lookup interpolation routine at 0x6080.

0x4110 — Engine Load Axis (Primary)

Field Value
Address 0x4110
Size 16 bytes
Format 16x1 unsigned 8-bit breakpoint table
Scaling units unknown
Duplicate 0x44E0 (identical contents, used by ignition)
Source historical

Contents:

03 06 09 0C 0F 12 15 18 1B 1E 21 24 27 2A 2D 30
(3, 6, 9, 12, 15, 18, 21, 24, 27, 30, 33, 36, 39, 42, 45, 48)

Evenly spaced with step 3.

Used by:

  • sub_7EC8 (fuel_map_lookup) at 0x7EEE — stored to 0x3076 as load axis for fuel table 0x4120
  • sub_CB52 (3-map selection) at 0xCB75 — stored to 0x3076 as load axis for maps 0x5200/0x5300/0x5400

Duplicate 0x44E0 used by:

  • sub_82D6 (ignition timing) at 0x82E2 and 0x8313 — stored to 0x3076 as load axis for timing maps

Load source: All three functions load engine load from 0x313C (16-bit), shift left 3 bits (ASL; ASL; ASL), and store to 0x3078 for the map lookup routine. The left-shift scales the raw load value to match the axis breakpoint format.

Notes

  • Two identical copies: 0x4100/0x4110 and 0x44D0/0x44E0 contain identical data. The fuel and 3-map subsystems use the primary copies; the ignition subsystem uses the duplicates. Having separate copies allows independent modification of axes per subsystem if needed — though in the stock and tuned ROMs they remain identical.
  • Shared interface: All three functions (fuel, ignition, 3-map) use the same map lookup workspace: 0x3072 (map pointer), 0x3074 (RPM axis pointer), 0x3076 (load axis pointer), 0x307A (current RPM), 0x3078 (current load shifted). They all call the same map_lookup_16x16 routine at 0x6080.
  • RPM scaling caveat: The historical analysis states "Real RPM = value x 12.5" giving a range of 200–1700 RPM across the 16 breakpoints. This seems low for a turbo engine that revs to 7000+ RPM. The x12.5 factor may apply to a different representation, or the axis may use a non-linear/inverse relationship since 0x3062 derives from a timer period (inversely proportional to RPM). Further investigation of the timer capture and scaling is needed.
  • Even spacing: Both axes are perfectly linear (step 8 for RPM, step 3 for load), which is common in ECU calibrations — it simplifies the interpolation arithmetic and provides uniform resolution across the operating range.
